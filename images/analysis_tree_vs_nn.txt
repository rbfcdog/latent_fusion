Why trees dominate on embeddings:
  ✓ Embeddings are pre-processed, dense representations
  ✓ Text embeddings (OpenAI) and time embeddings (MOMENT) 
    already capture semantic structure
  ✓ No further feature learning needed → trees excel at 
    axis-aligned splits in high-dimensional space
  ✓ Curse of dimensionality paradox: NNs struggle, 
    trees thrive with axis-aligned boundaries

Feature space characteristics:
  • Input dimension: 2560 (1536 text + 1024 time)
  • Embeddings are normalized, bounded
  • Patterns are linearly separable
  • Decision boundaries are simple

2. NUMERICAL ATTENTION PERFORMS TERRIBLY (-0.0872 R²)

Problems identified:
  ✗ Technical indicators have NaN values (771 NaNs filled)
  ✗ Extreme outliers: max value = 36,442,085
  ✗ Range: -100 to 36M (highly non-Gaussian)
  ✗ Standard scaling creates very large values in some dims
  ✗ NN learns poorly with such heterogeneous features
  ✗ Attention mechanism cannot handle feature instability

Solution:
  → Use embeddings WITHOUT numerical features
  → Or: Apply robust scaling, remove outliers, engineer 
     new numerical indicators
  → Numerical features add noise, not signal

  ✗ Numerical features with extreme outliers
  ✗ Attention mechanisms for this task

does NOT improve performance—it only adds complexity.
